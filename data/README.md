# Praxis Synthetic Attack Dataset

This dataset is the single repeatable incident Praxis investigates. It is
generated by `gen_scenario.py` and ingested into your live `index=main`.
There is no fake-data code path in the application - this data becomes
real, indexed Splunk events.

## Generate

```powershell
python data/gen_scenario.py
```

Output goes to `data/events/`:
- `praxis_auth.log`, `praxis_network.log`, `praxis_endpoint.log`, `praxis_egress.log` - one event per line, `<timestamp> key=value ...`, ready for `splunk add oneshot`
- `hec_events.jsonl` - same events in HEC JSON format, one per line

Timestamps are generated relative to "now", so **ingest soon after generating** - the planted campaign is anchored ~35 minutes before generation time.

## Schema

All sourcetypes use space-separated `key=value` pairs after a leading
`yyyy-MM-dd HH:mm:ss` timestamp. `data/props.conf` configures timestamp
and field extraction (`KV_MODE=auto` handles the rest automatically even
without props.conf).

### praxis:auth
| Field | Description |
|---|---|
| `user` | username |
| `app` | okta \| vpn \| o365 |
| `action` | login \| mfa_challenge |
| `status` | success \| denied \| approved |
| `auth_method` | password \| mfa_push \| mfa_totp |
| `src_ip`, `src_country`, `src_city` | login source |
| `host` | the user's workstation |
| `prev_country`, `prev_city`, `prev_login_minutes_ago`, `geo_velocity_kmh` | impossible-travel context (present on suspicious logins) |
| `mfa_attempt_number` | MFA fatigue sequence number |
| `travel_record` | corroborating travel-booking ID (false-alarm only) |

### praxis:network
| Field | Description |
|---|---|
| `user`, `src_host`, `dest_host` | who/what |
| `src_ip`, `dest_ip` | addresses |
| `protocol` | SMB \| RDP \| RPC \| HTTPS |
| `dest_port` | 445 \| 3389 \| 135 \| 443 |
| `dest_role` | file_server \| app_server \| web_server |
| `bytes_in`, `bytes_out` | transfer size |

### praxis:endpoint
| Field | Description |
|---|---|
| `user`, `host` | who/where |
| `action` | scheduled_task_created |
| `task_name`, `task_command`, `task_trigger` | task details |
| `parent_process` | spawning process |
| `signed` | true \| false - whether the binary is signed |
| `change_ticket` | approved change record (legitimate tasks only) |

### praxis:egress
| Field | Description |
|---|---|
| `user`, `src_host`, `src_ip`, `dest_ip`, `dest_domain` | who/where |
| `protocol` | HTTPS \| DNS |
| `bytes_out` | transfer size |
| `dest_reputation` | low \| high |
| `query_type`, `subdomain_entropy` | DNS-tunneling indicators |

Each stage alone trips only ONE low-severity alert; together they form a

## Ingest into Splunk (Windows, oneshot)

Run from an elevated PowerShell. Replace `<password>` with your admin password.

```powershell
$splunk = "C:\Program Files\Splunk\bin\splunk.exe"
$dataDir = "c:\Users\hp\Desktop\praxis\data\events"

& $splunk add oneshot "$dataDir\praxis_auth.log"     -index main -sourcetype praxis:auth     -auth admin:<password>
& $splunk add oneshot "$dataDir\praxis_network.log"  -index main -sourcetype praxis:network  -auth admin:<password>
& $splunk add oneshot "$dataDir\praxis_endpoint.log" -index main -sourcetype praxis:endpoint -auth admin:<password>
& $splunk add oneshot "$dataDir\praxis_egress.log"   -index main -sourcetype praxis:egress   -auth admin:<password>
```

### Optional: install props.conf and savedsearches.conf first

For correct timestamp/field extraction and the 5 alert definitions:

```powershell
Copy-Item data\props.conf          "C:\Program Files\Splunk\etc\system\local\props.conf"
Copy-Item data\savedsearches.conf  "C:\Program Files\Splunk\etc\apps\search\local\savedsearches.conf"
& "C:\Program Files\Splunk\bin\splunk.exe" restart
```

Then run the oneshot commands above.

## Verify Ingestion (SPL)

Run these in Splunk Search or via `splunk_run_query`:

```spl
| tstats count where index=main sourcetype IN (praxis:auth, praxis:network, praxis:endpoint, praxis:egress) by sourcetype
```
Expected: `praxis:auth=59`, `praxis:network=52`, `praxis:endpoint=52`, `praxis:egress=66`.

**Confirm the planted campaign:**
```spl
search index=main user=j.okonkwo earliest=-2h
| sort _time
| table _time sourcetype user host src_host dest_host action protocol dest_domain geo_velocity_kmh
```
Expected: 26 events (1 impossible travel + 6 MFA + 2 lateral + 1 persistence + 16 egress) across a ~22-minute span, all referencing `WKSTN-OKONKWO` and `FS01`.

**Confirm the 5 alerts fire** (after installing savedsearches.conf and waiting up to 5 min for the cron):
```spl
| rest /services/saved/searches splunk_server=local
| search title="Praxis - *"
| table title is_scheduled triggered_alert_count
```

**Confirm false alarms are present:**
```spl
search index=main (user=m.okafor OR (user=it.admin host=FS02)) earliest=-24h
| table _time sourcetype user geo_velocity_kmh travel_record signed change_ticket
```
